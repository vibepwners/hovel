Developing Mesh and Credential Providers

This guide is for developers implementing a Mesh provider, a standalone credential provider, optional listening-post control, or routing another Hovel module through a Mesh. The SDK defines contracts and dispatch; your provider owns its protocol, credential consumption, routing, task execution, listener runtime, and provider-side durable state.

Start with the boundary

Do not implement no-op methods to satisfy a large interface. In Go, implement MeshDescriber and only the optional provider interfaces you need. In Python and Rust, override only the Mesh hooks you support; defaults fail explicitly. Advertise the same surface in mesh.describe.

Declare credential delivery only when you consume it

A Mesh provider can advertise an optional, versioned credentialDelivery descriptor. This is discovery metadata, not credential material: it contains slot constraints and supported delivery modes, never certificates, private keys, passwords, or provider config. Omit the object when the provider does not consume Hovel-managed credentials.

Prefer stamp-standard with a stable named slot. Add stamp-advanced only when an operator must explicitly select an offset, virtual address, symbol, marker, pattern, or versioned provider-defined target. A capability is a promise: do not advertise a mode unless the provider can validate and execute its full contract. 

{
  "credentialDelivery": {
    "schemaVersion": "hovel.pki.credential-delivery/v1",
    "deliveryCapabilities": ["runtime", "stamp-standard"],
    "credentialSlots": [
      {
        "name": "control-plane-mtls",
        "purpose": "mtls-server",
        "endpointRole": "server",
        "consumerType": "mesh-listener",
        "acceptedBundleVersions": ["hovel.pki.bundle/v1"],
        "acceptedProfiles": ["mtls-server"],
        "acceptedCompatibilityTargets": ["portable-x509"],
        "acceptedProjections": ["bundle"],
        "acceptedMaterialForms": ["private-bytes"],
        "maximumEncodedBytes": 16384,
        "remainderPolicy": "preserve",
        "privateMaterial": "allowed"
      }
    ],
    "stampTargetKinds": ["named-slot"]
  }
}

Private material is write-only operation input. Descriptor, topology, listener, beacon, module-list, and normal inspect responses must not echo resolved bundles or secrets. Hovel records assignment, descriptor, target, material-reference, artifact, and digest provenance without persisting the replacement bytes in a credential stamp.

Select assignments from a front end

A front end never resolves a certificate or private key itself. Bind and activate assignments first, then add non-secret credentials selectors to StartMeshListener, RunMeshTask, OpenMeshStream, or OpenMeshBridge. The daemon validates the active assignment, its consumer, the provider's current descriptor, and the requested slot before starting the provider hook.

Assignment consumer IDs use the provider's canonical name without its @version suffix. Use mesh-provider with <provider-name>, mesh-listener with <provider-name>/<listener-id>, or mesh-node with <provider-name>/<node-id>. Hovel derives those subjects from the operation; callers cannot substitute an unrelated consumer.

The first selector path is deliberately runtime-only and direct: use certificate-der/public, public-key-spki/public, or private-key-pkcs8/private-bytes. Declare separate slots when a provider needs both a certificate and a private key. File materialization, provider encoding, and stamping remain available through their dedicated contracts; they are not silently approximated by this API. 

{
  "moduleId": "mesh-provider@1.2.3",
  "request": {
    "listenerId": "edge",
    "name": "edge HTTPS listener"
  },
  "credentials": [
    {
      "requestId": "edge-listener-cert-v1",
      "assignmentId": "assignment-edge-listener",
      "slotName": "tls-certificate",

      "slotName": "tls-private-key",
      "capability": "runtime",
      "material": {"projection": "private-key-pkcs8", "form": "private-bytes"}

}
Retry with the same identity. Keep each credential requestId, the credential-context operationId, and the operation inputs unchanged when retrying an uncertain response. Reusing a request ID with a different assignment, slot, provider version, descriptor, or operation scope fails with an idempotency conflict. A completed retry returns only durable, secret-free receipt evidence and does not rerun the provider hook.

For a credential-bearing call, Hovel launches one provider process, repeats handshake and credential.describe in that process, verifies the exact module ID, provider version, and descriptor digest, delivers every selected credential, and only then invokes the Mesh method. Any mismatch fails before the first secret is delivered. Resolved bytes are cleared after the Mesh call returns and never appear in daemon RPC responses or Mesh bookkeeping.

Implement the strongly typed credential provider hooks

All three SDKs dispatch the same five optional methods. Go expresses execution hooks as small interfaces. Python and Rust provide default methods that fail explicitly until overridden. Do not override a hook without advertising its matching capability or schema, and do not advertise a capability without a real implementation.

Go: complete credential capability implementation

This type assumes the ordinary hovel.Module metadata and Run methods are implemented elsewhere. The maps model ephemeral provider state that the next Mesh method in the same process can consume. A real provider should replace them with its native loader and deployment API. 

type CredentialMesh struct {
    runtime     map[string][]byte
    files       map[string][]string
    deployments map[string][]byte
}

func NewCredentialMesh() *CredentialMesh {
    return &CredentialMesh{
        runtime: map[string][]byte{}, files: map[string][]string{},
        deployments: map[string][]byte{},
    }
}

func credentialDescriptor() hovel.CredentialDeliveryDescriptor {
    return hovel.CredentialDeliveryDescriptor{
        SchemaVersion: hovel.CredentialDeliverySchemaV1,
        Capabilities: []hovel.CredentialDeliveryCapability{
            hovel.CredentialDeliveryRuntime,
            hovel.CredentialDeliveryFiles,
            hovel.CredentialDeliveryStampStandard,
        },
        Slots: []hovel.CredentialSlot{{
            Name: "control-plane-mtls",
            Purpose: hovel.CredentialPurposeMTLSServer,
            EndpointRole: hovel.CredentialEndpointServer,
            ConsumerType: hovel.CredentialConsumerMeshListener,
            AcceptedBundleVersions: []string{"hovel.pki.bundle/v1"},
            AcceptedProfiles: []string{"mtls-server"},
            AcceptedCompatibilityTargets: []string{"portable-x509"},
            AcceptedProjections: []hovel.CredentialProjection{hovel.CredentialProjectionBundle},
            AcceptedMaterialForms: []hovel.CredentialMaterialForm{hovel.CredentialMaterialPrivateBytes},
            MaximumEncodedBytes: 16 * 1024,
            RemainderPolicy: hovel.CredentialStampRemainderPreserve,
            PrivateMaterial: hovel.CredentialPrivateMaterialAllowed,
        }},
        StampTargetKinds: []hovel.CredentialStampTargetKind{
            hovel.CredentialStampTargetNamedSlot,
        },
        ProviderEncodingSchemas: []hovel.CredentialProviderEncodingSchema{{
            ProviderID: "example-device", SchemaVersion: "example.device-bundle/v1",
            AcceptedSourceProjections: []hovel.CredentialProjection{hovel.CredentialProjectionBundle},
            AcceptedSourceForms: []hovel.CredentialMaterialForm{hovel.CredentialMaterialPrivateBytes},
            OutputForms: []hovel.CredentialMaterialForm{hovel.CredentialMaterialPrivateBytes},
        }},
    }
}

func (*CredentialMesh) DescribeCredentialDelivery() (hovel.CredentialDeliveryDescriptor, error) {
    return credentialDescriptor(), nil
}

func (p *CredentialMesh) LoadRuntimeCredential(
    req hovel.CredentialRuntimeRequest,
) (hovel.CredentialDeliveryReceipt, error) {
    material, ok := req.Material.Bytes()
    if !ok { return hovel.CredentialDeliveryReceipt{}, errors.New("runtime slot requires bytes") }
    p.runtime[req.SlotName] = material // defensive copy returned by Bytes
    return hovel.CredentialDeliveryReceipt{
        RequestID: req.RequestID,
        ProviderReference: "runtime-slot:" + req.AssignmentID,
    }, nil
}

func (p *CredentialMesh) LoadCredentialFiles(
    req hovel.CredentialFilesRequest,
) (hovel.CredentialDeliveryReceipt, error) {
    paths := make([]string, len(req.Files))
    for i, file := range req.Files { paths[i] = string(file.Path) }
    p.files[req.SlotName] = paths
    return hovel.CredentialDeliveryReceipt{
        RequestID: req.RequestID,
        ProviderReference: "file-slot:" + req.AssignmentID,
    }, nil
}

func (*CredentialMesh) EncodeCredentialMaterial(
    req hovel.CredentialEncodingRequest,
) (hovel.CredentialEncodingResult, error) {
    encoded, ok := req.Source.Bytes()
    if !ok { return hovel.CredentialEncodingResult{}, errors.New("encoder requires bytes") }
    if uint64(len(encoded)) > req.MaximumEncodedBytes {
        return hovel.CredentialEncodingResult{}, errors.New("encoded credential exceeds bound")
    }
    return hovel.CredentialEncodingResult{
        RequestID: req.RequestID, Form: req.OutputForm,
        Encoding: req.ProviderSchema, SHA256: req.Source.SHA256,
        Data: hovel.CredentialBytes(encoded),
    }, nil
}

func (p *CredentialMesh) StampCredential(
    req hovel.CredentialStampExecutionRequest,
) (hovel.CredentialStampExecutionResult, error) {
    material, ok := req.Material.Bytes()
    if !ok { return hovel.CredentialStampExecutionResult{}, errors.New("deployment requires bytes") }
    p.deployments[req.StampID] = material
    output, err := hovel.NewCredentialStampDeploymentOutput(hovel.CredentialDeploymentOutput{
        Reference: "deployment:" + req.StampID,
        Receipt: hovel.CredentialBytes([]byte("accepted:" + req.StampID)),
    })
    if err != nil { return hovel.CredentialStampExecutionResult{}, err }
    return hovel.CredentialStampExecutionResult{
        StampID: req.StampID, Output: output,
        TargetResolution: hovel.CredentialStampTargetUnchanged,
        ResolvedTarget: req.Request.Target,
        BytesWritten: hovel.CredentialCanonicalUint64(strconv.FormatUint(req.Request.EncodedBytes, 10)),
        MaterialDigests: append([]hovel.CredentialStampedMaterialDigest(nil), req.ExpectedDigests...),
    }, nil
}

Python: complete credential capability implementation

class CredentialMesh(HovelModule):
    name = "credential-mesh"
    version = "v0.1.0"
    module_type = "payload_provider"

    def __init__(self) -> None:
        self._runtime: dict[str, bytes] = {}
        self._files: dict[str, list[str]] = {}
        self._deployments: dict[str, bytes] = {}

    def run(self, _ctx: Context) -> Result:
        # Credential operations use the typed hooks below; run remains a
        # harmless module-protocol fallback.
        return Result.ok(summary="credential provider ready")

    def describe_credential_delivery(self) -> CredentialDeliveryDescriptor:
        return CredentialDeliveryDescriptor(
            capabilities=[
                CredentialDeliveryCapability.RUNTIME,
                CredentialDeliveryCapability.FILES,
                CredentialDeliveryCapability.STAMP_STANDARD,
            ],
            slots=[CredentialSlot(
                name="control-plane-mtls",
                purpose=CredentialPurpose.MTLS_SERVER,
                endpoint_role=CredentialEndpointRole.SERVER,
                consumer_type=CredentialConsumerType.MESH_LISTENER,
                accepted_bundle_versions=["hovel.pki.bundle/v1"],
                accepted_profiles=["mtls-server"],
                accepted_compatibility_targets=["portable-x509"],
                accepted_projections=[CredentialProjection.BUNDLE],
                accepted_material_forms=[CredentialMaterialForm.PRIVATE_BYTES],
                maximum_encoded_bytes=16 * 1024,
                remainder_policy=CredentialStampRemainderPolicy.PRESERVE,
                private_material=CredentialPrivateMaterialPolicy.ALLOWED,
            )],
            stamp_target_kinds=[CredentialStampTargetKind.NAMED_SLOT],
            provider_encoding_schemas=[CredentialProviderEncodingSchema(
                provider_id="example-device",
                schema_version="example.device-bundle/v1",
                accepted_source_projections=[CredentialProjection.BUNDLE],
                accepted_source_forms=[CredentialMaterialForm.PRIVATE_BYTES],
                output_forms=[CredentialMaterialForm.PRIVATE_BYTES],
            )],
        )

    def load_runtime_credential(self, request: CredentialRuntimeRequest) -> CredentialDeliveryReceipt:
        if not isinstance(request.material.value, CredentialBytes):
            raise ValueError("runtime slot requires bytes")
        self._runtime[request.slot_name] = request.material.value.value
        return CredentialDeliveryReceipt(
            request_id=request.request_id,
            provider_reference=f"runtime-slot:{request.assignment_id}",
        )

    def load_credential_files(self, request: CredentialFilesRequest) -> CredentialDeliveryReceipt:
        self._files[request.slot_name] = [item.path for item in request.files]
        return CredentialDeliveryReceipt(
            request_id=request.request_id,
            provider_reference=f"file-slot:{request.assignment_id}",
        )

    def encode_credential_material(self, request: CredentialEncodingRequest) -> CredentialEncodingResult:
        if not isinstance(request.source.value, CredentialBytes):
            raise ValueError("encoder requires bytes")
        encoded = request.source.value.value
        if len(encoded) > request.maximum_encoded_bytes:
            raise ValueError("encoded credential exceeds bound")
        return CredentialEncodingResult(
            request_id=request.request_id, form=request.output_form,
            encoding=request.provider_schema, sha256=request.source.sha256,
            data=encoded,
        )

    def stamp_credential(self, request: CredentialStampExecutionRequest) -> CredentialStampExecutionResult:
        if not isinstance(request.material.value, CredentialBytes):
            raise ValueError("deployment requires bytes")
        self._deployments[request.stamp_id] = request.material.value.value
        return CredentialStampExecutionResult(
            stamp_id=request.stamp_id,
            output=CredentialDeploymentOutput(
                reference=f"deployment:{request.stamp_id}",
                receipt=f"accepted:{request.stamp_id}".encode(),
            ),
            target_resolution=CredentialStampTargetResolution.UNCHANGED,
            resolved_target=request.request.target,
            bytes_written=str(request.request.encoded_bytes),
            material_digests=list(request.expected_digests),
        )

Rust: complete credential capability implementation

#[derive(Default)]
struct CredentialMesh {
    runtime: RefCell<BTreeMap<String, Vec<u8>>>,
    files: RefCell<BTreeMap<String, Vec<String>>>,
    deployments: RefCell<BTreeMap<String, Vec<u8>>>,
}

impl Module for CredentialMesh {
    fn info(&self) -> Info {
        Info {
            name: "credential-mesh".into(), version: "v0.1.0".into(),
            module_type: ModuleType::PayloadProvider,
            summary: "credential-aware Mesh provider".into(),
            description: String::new(),
            tags: vec!["mesh".into(), "credential-provider".into()],
            discovery_context: Vec::new(),
        }
    }

    fn schema(&self) -> Schema { Schema::default() }

    fn run(&self, _ctx: &mut Context) -> Outcome {
        // Credential operations use the typed hooks below; run remains a
        // harmless module-protocol fallback.
        Outcome::ok(Vec::new()).with_summary("credential provider ready")
    }

    fn describe_credential_delivery(&self) -> Result<CredentialDeliveryDescriptor, String> {
        Ok(CredentialDeliveryDescriptor {
            capabilities: vec![
                CredentialDeliveryCapability::Runtime,
                CredentialDeliveryCapability::Files,
                CredentialDeliveryCapability::StampStandard,
            ],
            slots: vec![CredentialSlot {
                name: "control-plane-mtls".into(),
                purpose: CredentialPurpose::MtlsServer,
                endpoint_role: CredentialEndpointRole::Server,
                consumer_type: CredentialConsumerType::MeshListener,
                accepted_bundle_versions: vec!["hovel.pki.bundle/v1".into()],
                accepted_profiles: vec!["mtls-server".into()],
                accepted_compatibility_targets: vec!["portable-x509".into()],
                accepted_projections: vec![CredentialProjection::Bundle],
                accepted_material_forms: vec![CredentialMaterialForm::PrivateBytes],
                maximum_encoded_bytes: 16 * 1024,
                remainder_policy: CredentialStampRemainderPolicy::Preserve,
                private_material: CredentialPrivateMaterialPolicy::Allowed,
            }],
            stamp_target_kinds: vec![CredentialStampTargetKind::NamedSlot],
            provider_encoding_schemas: vec![CredentialProviderEncodingSchema {
                provider_id: "example-device".into(),
                schema_version: "example.device-bundle/v1".into(),
                accepted_source_projections: vec![CredentialProjection::Bundle],
                accepted_source_forms: vec![CredentialMaterialForm::PrivateBytes],
                output_forms: vec![CredentialMaterialForm::PrivateBytes],
            }],
            ..CredentialDeliveryDescriptor::default()
        })
    }

    fn load_runtime_credential(
        &self, req: CredentialRuntimeRequest,
    ) -> Result<CredentialDeliveryReceipt, String> {
        let material = req.material.bytes().ok_or("runtime slot requires bytes")?.to_vec();
        self.runtime.borrow_mut().insert(req.slot_name, material);
        Ok(CredentialDeliveryReceipt {
            request_id: req.request_id,
            provider_reference: Some(format!("runtime-slot:{}", req.assignment_id)),
            receipt_sha256: None,
        })
    }

    fn load_credential_files(
        &self, req: CredentialFilesRequest,
    ) -> Result<CredentialDeliveryReceipt, String> {
        let paths = req.files.iter().map(|item| item.path.expose().to_string()).collect();
        self.files.borrow_mut().insert(req.slot_name, paths);
        Ok(CredentialDeliveryReceipt {
            request_id: req.request_id,
            provider_reference: Some(format!("file-slot:{}", req.assignment_id)),
            receipt_sha256: None,
        })
    }

    fn encode_credential_material(
        &self, req: CredentialEncodingRequest,
    ) -> Result<CredentialEncodingResult, String> {
        let encoded = req.source.bytes().ok_or("encoder requires bytes")?.to_vec();
        if encoded.len() as i64 > req.maximum_encoded_bytes {
            return Err("encoded credential exceeds bound".into());
        }
        Ok(CredentialEncodingResult {
            request_id: req.request_id, form: req.output_form,
            encoding: req.provider_schema, sha256: req.source.sha256,
            data: CredentialBytes::new(encoded)?,
        })
    }

    fn stamp_credential(
        &self, req: CredentialStampExecutionRequest,
    ) -> Result<CredentialStampExecutionResult, String> {
        let material = req.material.bytes().ok_or("deployment requires bytes")?.to_vec();
        self.deployments.borrow_mut().insert(req.stamp_id.clone(), material);
        let expected_digests = req.expected_digests;
        Ok(CredentialStampExecutionResult {
            stamp_id: req.stamp_id.clone(),
            output: CredentialStampOutput::Deployment(CredentialDeploymentOutput {
                reference: format!("deployment:{}", req.stamp_id),
                receipt: CredentialBytes::new(format!("accepted:{}", req.stamp_id).into_bytes())?,
            }),
            target_resolution: CredentialStampTargetResolution::Unchanged,
            resolved_target: req.request.target,
            bytes_written: req.request.encoded_bytes.to_string(),
            material_digests: expected_digests,
        })
    }
}
Return evidence, not secrets. Runtime/file receipts contain only the matching request id, an optional non-secret provider reference, and an optional receipt digest. Encoding and stamp results necessarily carry transient output bytes, protected paths, or deployment receipts; never log them. Hovel validates and ingests them, then persists only non-secret metadata and digests.

Wire provider delivery into its consumer

Loading a credential in one short-lived module invocation and starting a listener in another loses in-memory state. The implemented runner therefore starts one process, sends all validated credential.runtime and credential.files requests, records their receipts, and then calls the consuming mesh.listener.start, mesh.task, or mesh.open_stream before shutting down or adopting the session. 

active Assignment + exact CredentialDeliveryDescriptor
  -> resolve permitted projection and material form
  -> build CredentialOperationDeliveries for the same module and slot
  -> credential.runtime / credential.files in one module process
  -> consuming Mesh operation in that same process
  -> durable execution plan + non-secret receipt/digests

The assignment/domain validation and same-process runner path are implemented. The production daemon does not yet derive these deliveries from an assignment ID, and current external Mesh requests have no assignment field. Credential execution and stamp daemon endpoints are therefore list/inspect-only today; provider authors can implement and contract-test the SDK surface now without claiming that an operator can initiate the complete flow.

1. Describe capabilities without side effects

mesh.describe is catalog metadata. It must be cheap, deterministic, offline, and side-effect free. Put dynamic node state in topology, beacon, or listener calls instead. A provider supporting routed streams, destination tasking, and an HTTPS listening post might return:

      "readOnly": true,

2. Report topology, beacons, and triggers

Nodes are operator-addressable Mesh participants. Destinations are ordinary hosts or services reachable through a node or route. Do not invent a node for every destination. Use listenerId on nodes, beacons, and triggers when provenance matters. 

{
  "root": "controller",
  "nodes": [
    { "id": "controller", "kind": "controller", "state": "online" },
    {
      "id": "relay-west",
      "parentId": "controller",
      "listenerId": "listener-west",
      "kind": "relay",
      "state": "online"
    }
  ],
  "links": [
    { "id": "controller-west", "source": "controller", "target": "relay-west", "state": "up" }
  ],
  "routes": [
    { "id": "route-west", "nodes": ["controller", "relay-west"] }
  ]
}

Triggers are declarations in the descriptor; beacons are time-stamped runtime observations. Neither starts work by itself or bypasses Hovel's planning and confirmation path.

3. Implement tasking

Validate the task kind and required target scope before contacting the target. Echo correlation and routing fields in the result when they are known. For a copy-then-run implant flow, use upload_execute; reserve load for a provider-native in-memory or protocol-specific loader. 

{
  "runId": "run-lab-1",
  "taskId": "task-deploy-1",
  "kind": "upload_execute",
  "nodeId": "relay-west",
  "listenerId": "listener-west",

Provider-native command, survey, and delivery tasks belong here. If an existing module can already operate through a TCP or UDP endpoint, prefer the bridge workflow below instead of duplicating that module inside the provider.

4. Implement routed streams

mesh.open_stream returns a normal SessionRef. The provider implements session read, write, and close against its own route. The daemon can then expose the session through ordinary session APIs or a local Mesh Bridge.

  • For TCP-like flows, preserve byte-stream semantics.
  • For UDP-like flows, advertise datagram and preserve exactly one datagram per non-empty session read or write.
  • For ICMP, GRE, raw IP, or another non-socket protocol, keep it provider-defined until a concrete consumer requires a generic packet adapter.
  • Return an error for unsupported protocols; do not silently reinterpret them as TCP.

5. Add listening-post lifecycle only when needed

Listener listing is read-only. Start and stop are idempotent mutations using a caller-selected stable ID. IDs are unique only inside one provider Mesh; daemon-wide identity is (moduleId, listenerId).

The result deliberately has no config. Treat start configuration as write-only, keep it out of listener read models, errors, and logs, and never store credentials directly in daemon bookkeeping. Prefer references to a provider-owned secret store. A provider-managed listener must survive the short-lived module RPC invocation; use a separate service, supervisor, or another durable provider-owned runtime.

Current bookkeeping is live-daemon state. Mesh operation records support external control and inspection while the daemon is running, but the current MeshBook retains at most 4,096 recent operations in memory and is cleared by a daemon restart. Provider-owned listeners and nodes may continue running; reconcile them from provider discovery rather than treating the live ledger as durable truth.

Using a Mesh with another module

The clean integration is authenticated endpoint substitution: the front end completes the normal throw safeguards, asks the daemon for a bridge, and gives the returned loopback endpoint and one-time bridge capability to an SDK-aware consumer. The consumer does not need to understand the provider's route.

Mesh mutation calls are privileged primitives. Listener, task, stream, and bridge mutations are available only over the owner-controlled Unix socket. Loopback TCP is read-only and rejects privileged calls. The primitives do not synthesize a throw plan or confirmation; front ends must complete those safeguards before opening a bridge or starting an execution-capable task.
  1. Inspect the Mesh and choose a node or route.
  2. Create, review, and confirm the normal persisted throw plan.
  3. Call OpenMeshBridge over the privileged Unix socket.
  4. Connect to localHost/localPort and authenticate with the returned capability.
  5. Run the consumer module against that authenticated connection.
  6. Close the bridge and inspect ListMeshOperations.
POST /hovel.daemon.v1.DaemonService/OpenMeshBridge
Content-Type: application/json

{
  "moduleId": "lab-relay-mesh@v0.1.0",
  "request": {
    "runId": "run-lab-1",
    "nodeId": "relay-west",
    "listenerId": "listener-west",
    "destinationHost": "192.0.2.25",
    "destinationPort": 443,
    "protocol": "tcp"
  }
}
{
  "operationId": "mesh-op-43",
  "sessionId": "run-lab-1-session-1",
  "localHost": "127.0.0.1",
  "localPort": 43117,
  "localAddress": "127.0.0.1:43117",
  "capability": "uL8wGvVukVynTng2uR6U4oqxm8v0cMxfR7VaJkW_1eM"
}
The capability is a bearer secret. Do not log, persist, or cache it; the response carries Cache-Control: no-store. For TCP, send the capability followed by a newline as the first bytes on the connection. For UDP, send it as one complete authentication datagram before application datagrams. Missing or incorrect capabilities are rejected without claiming the bridge.

A web, Elixir, CLI, or other control plane can configure an SDK-aware consumer with target.host=127.0.0.1, target.port=43117, and the in-memory capability. A standard SDK dial helper can perform the authentication preface so the module's protocol logic still sees an ordinary connected stream or datagram endpoint. The daemon bridge moves bytes and the Mesh provider owns the remote route. Open one bridge per independent TCP connection or UDP peer association. 

endpoint, err := hovel.NewMeshBridgeEndpoint(
    bridge.LocalHost,
    bridge.LocalPort,
    hovel.MeshBridgeCapability(bridge.Capability),
)
if err != nil {
    return err
}
connection, err := hovel.DialMeshBridge(ctx, hovel.MeshBridgeNetworkTCP, endpoint)
if err != nil {
    return err
}
defer connection.Close()

Python provides MeshBridgeEndpoint and connect_mesh_bridge; Rust provides MeshBridgeCapability, MeshBridgeEndpoint, and connect_mesh_bridge_tcp/connect_mesh_bridge_udp. Each helper validates the loopback endpoint and bearer capability before returning an ordinary connected socket.

Use RunMeshTask instead when the provider itself owns the operation (for example, provider-native survey, command, upload-and-execute, or load). Use a bridge when reusing a protocol client or module that already knows how to talk to a host and port. These approaches can coexist in one provider.

Testing checklist

  • Descriptor calls are deterministic and perform no target or listener I/O.
  • Every advertised task kind and target scope has a success and rejection test.
  • Unsupported optional methods fail explicitly.
  • Topology IDs are stable; routes reference existing nodes and links.
  • Listener start/stop reject blank IDs before provider side effects and return the exact requested ID.
  • Repeated listener start/stop calls are safe and converge on the same state.
  • Listener configuration and credentials never appear in results, errors, logs, or daemon operation records.
  • Stream tests cover read, write, close, cancellation, and datagram boundaries when applicable.
  • Bridge tests bind only loopback and close sessions on failure.
  • Task, stream, listener, and bridge calls appear in ListMeshOperations with provider and listener provenance.
