Hovel Modules

Modules have their own documentation space separate from the core book. Each module space should document what it provides, how to build and test it, how to operate it from Hovel, and where its source-level reference material lives.

Module Spaces

Index Catalogue All integrated modules with type, status, and guide links. Payload Provider picblobs Position-independent payload blob provider integrated from the original pic repo.

Documentation Contract

A module space should contain a landing page, user guide, build notes, test notes, and reference page. Source-specific historical material can remain inside the module tree, but operator-facing docs should use this site structure.
