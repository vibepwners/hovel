Testing Squatter

The standard module gate runs host-side Go tests, provider contract tests, package/install checks, PE inspection, and source-contract tests. Wine-backed runtime tests are separated because they need a Windows execution environment.

Primary Checks

task modules:test
task bazel:test -- //:modules_test

This gate covers the provider, shell, SMB pipe client, wire framing, Squatter package checks, workspace module-install checks, PE inspection, and source layout contracts.

Wine Runtime Checks

task modules:wine-test
task bazel:test -- --platforms=//platforms:windows_x86 --extra_toolchains=@mingw_i686//:toolchain --run_under=wine //:modules_wine_test

The Wine suite runs C/C++ integration checks for command execution, echo, file transfer, frames, framing, nanopb round trips, and the GoogleTest smoke path. The manual functional test launches the real payload under Wine and drives it through the Go client protocol. 

task bazel:test -- //:modules_functest

Static and Shape Checks

PE inspection verifies the produced payload shape. Source-contract tests protect the expected Windows payload layout. Manual static checks for the C tree live under modules/squatter/windows/src. 

task bazel:test -- //modules/squatter/windows:squatter_x86_exe_pe_test
task bazel:test -- //modules/squatter/windows:source_contract_test
task bazel:build -- //modules/squatter/windows/src:static_analysis
