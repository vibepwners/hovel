Getting Started

Prerequisites

  • Bazel 9.1.1 (see .bazelversion)
  • QEMU user-mode for cross-architecture testing — either binfmt_misc handlers (qemu-user-binfmt, preferred) or standalone qemu-*-static interpreters
  • Python 3.10+
  • uv (recommended) or pip/venv
  • clang-tidy for full C lint (pre-push / tools/c_lint_check.sh)

Toolchains are fetched automatically via Bootlin for Linux cross-compilation, including x86, ARM, AArch64, MIPS, s390x, SPARC, PowerPC, ppc64le, and RISC-V targets.

Quick start

# Set up Python environment
source sourceme

# Build and stage all blobs + runners
task stage

# Verify everything works
picblobs-cli verify

# Run the full test suite
task test

source sourceme also installs the repo's Git hooks with lefthook, so local formatting and lint checks start running automatically on git commit and git push.

Docker / Podman (no local setup)

A Fedora 43 dev container with all dependencies pre-installed is provided in ci/:

ci/dev.sh                                      # interactive shell
ci/dev.sh 'source sourceme && picblobs-cli verify'  # run a command and exit
